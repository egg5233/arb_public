| Browser → /api/config POST | Untrusted JSON body crosses HTTP boundary; bearer-token auth gate before this handler |
| /api/config handler → config.json | File-system mutation gated by validation + active-position guard |
| /api/config handler → Redis (pg:positions:active) | Read-only lookup for delete-safety check |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | T (Tampering) | priceGapUpdate.Candidates payload | mitigate | validatePriceGapCandidates rejects out-of-range numerics, regex-bounded symbol, exchange enum membership; pointer-to-slice prevents nil-vs-empty confusion (Pitfall 2) |
| T-10-02 | T (Tampering) | Symbol case/whitespace drift | mitigate | strings.ToUpper + TrimSpace before regex check + before storage (Pitfall 3) — prevents tracker tuple-drift bug |
| T-10-03 | I (Information disclosure via injection) | error envelope echoes user input | accept | Validation errors include candidate index + raw symbol; same risk surface as existing /api/config error responses; no PII; logs already redacted by writeJSON envelope |
| T-10-04 | E (Elevation: bypass active-position safety) | candidate removal while position live | mitigate | guardActivePositionRemoval compares prev∖next tuple set against pg:positions:active before mutation; HTTP 409 on conflict; covers tuple-change orphan path (Pitfall 4) |
| T-10-05 | D (Denial of service) | Large candidates array exhausts memory | accept | Reasonable cap implicit from MaxPositionUSDT (≤50000) and operator workflow (≤12 candidates expected pre-Phase 12 per PriceGapMaxCandidates); unbounded array DoS already possible on /api/config Risk* fields and not in Phase 10 scope |
| T-10-06 | E (Elevation: missing auth on POST) | unauthenticated client mutates candidates | mitigate | Existing isMutatingEndpoint gate covers POST /api/config — Phase 9 T-09-06 hardening (D-23 in 09-CONTEXT.md). No new auth code needed; verified in handlers.go middleware chain |
| T-10-07 | R (Repudiation) | who edited the candidate list | accept | No audit log in Phase 10 (deferred to v3.0 per CONTEXT "Deferred Ideas"); single-operator system per CLAUDE.local.md |
| T-10-08 | I (Race / IDOR via tuple-change) | edit changes tuple, old position orphaned | mitigate | guardActivePositionRemoval compares prev∖next tuple set, blocks at 409 (Pitfall 4 explicit case + Test 12) |
</threat_model>
