# Phase 11 — UI Design Contract

> Visual and interaction contract for the Discovery sub-section added inside the existing Price-Gap dashboard tab. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Project tech: React 18 + Vite + Tailwind v4 + Recharts, embedded into Go binary via `go:embed`. **Theme-scoped tokens are already in place at `web/src/index.css`** (Binance dark `data-theme="new"` + Tailwind classic). This spec inherits and reuses those tokens — no new design system.

## Spacing Scale

Reuses Tailwind v4 default spacing (multiples of 4 = 0.25rem). The Discovery sub-section uses these tokens exclusively:

| Token | Tailwind class | Pixels | Usage in Discovery section |
|-------|---------------|--------|----------------------------|
| xs | `gap-1` / `p-1` | 4px | Icon-to-text gaps inside pills/badges |
| sm | `gap-2` / `p-2` / `mb-2` | 8px | Inline label-to-value spacing; in-card row spacing; existing `mb-2` in PriceGap section header |
| md | `gap-4` / `p-4` | 16px | **Card interior padding** (matches existing `panelCls` pattern in PriceGap.tsx); column gaps inside cycle-stats grid |
| lg | `gap-6` / `p-6` / `space-y-6` | 24px | **Card-to-card vertical spacing** within the Discovery sub-section |
| xl | `space-y-8` | 32px | **Major section break** between Discovery sub-section and the existing PriceGap sections (Master toggle / Candidates / Positions) — mirrors existing `space-y-8` on the page root |
| 2xl | `mt-12` (rare) | 48px | Reserved for the Phase-12 timeline placeholder block bottom margin (visually marks "Phase 12 will fill below") |

**Exceptions:**
- Sub-cards inside the cycle-stats grid use `p-3` (12px) to match the existing tighter density of stat tiles in PriceGap.tsx.
- Empty placeholder card uses `p-8` (32px) for the centered empty-state copy.

**Source:** verified against `web/src/pages/PriceGap.tsx` `panelCls` (existing convention is `bg-gray-900 border border-gray-800 rounded-lg p-4`).

## Typography

Phase 11 uses **3 sizes + 1 mono variant** (4 roles total) and **2 weights** (400 + 700). All sizes already exist in Tailwind defaults; no new font metrics introduced.

| Role | Tailwind class | Size | Weight | Line height | Usage |
|------|---------------|------|--------|-------------|-------|
| Body | `text-sm` | 14px | `font-normal` (400) | 1.5 (1.25rem default) | Card descriptive copy, table rows, why-rejected reason labels |
| Label | `text-xs` | 12px | `font-normal` (400) | 1.5 | Stat tile sub-labels ("Last run", "Candidates seen"), tooltip text, table column headers |
| Heading | `text-base` | 16px | `font-bold` (700) | 1.5 | Card titles ("Cycle Stats", "Score History", "Why Rejected", "Promote/Demote Timeline") — matches existing `sectionTitle` in PriceGap.tsx |
| Display (numeric) | `text-lg` + `font-mono tabular-nums` | 18px | `font-bold` (700) | 1.2 | Big stat values inside cycle-stats card (`candidates_seen`, `accepted`, `rejected`, `errors`) |

**Mono numeric column rule:** every numeric value in a card or table uses `font-mono tabular-nums` so columns align — verified against existing `tabular-nums` usage at `PriceGap.tsx:822` and the `font-feature-settings: "tnum" 1` declaration in `index.css:200`.

**Bilingual rule:** `font-bold` is consistent across en + zh-TW (Traditional Chinese renders correctly with the same weight; no font-family override needed for zh-TW).

The Discovery sub-section inherits the **existing dashboard theme palette** — no new colors introduced. The 60/30/10 split already exists in `web/src/index.css` and is reused as-is.

| Role | Theme=new (Binance) | Theme=classic (Tailwind) | Usage |

| Positive | `#0ecb81` (green-400) | `#34d399` (green-400) | Score-history line stroke, "Scanner ON" indicator, accepted-count value |

### Accent reserved-for list (10% rule)

The yellow/blue accent is reserved EXCLUSIVELY for these Discovery elements — never used as a generic interactive color:

1. **Score-history threshold band** — `<ReferenceArea>` overlay above `PriceGapAutoPromoteScore` rendered with `fill="#f0b90b"` at `fillOpacity={0.10}` (or blue-500 at the same opacity in classic).
2. **"Scanner ON" pill indicator** in the Discovery section header (only when `PriceGapDiscoveryEnabled=true`). Inactive uses gray-700.
3. **Threshold-band legend swatch** in the score-history card (small 12px square next to the legend label).
4. **Focus ring on the section's only interactive control** — the Score History symbol-selector dropdown — inherits the existing `outline: 2px solid #f0b90b` from `index.css:230–231`.

**Forbidden accent uses:** generic links, table row hover, button defaults (the section has no buttons; selection is via dropdown), placeholder-card border (uses gray-800).

| State | Theme=new color | Where it appears |

| Scanner ERRORED | `bg-[#f6465d]/15 text-[#f6465d] border-[#f6465d]/40` | Banner shown when last cycle's `errors > 0` OR `cycle_failed=true` for ≥1 cycle (mirrors existing `colorMapNew.error` from StatusBadge.tsx) |

### Placement

The Discovery sub-section is **inserted as a new section inside `web/src/pages/PriceGap.tsx`**, between the Master-toggle row (existing) and the Candidates table (existing). It uses the same `space-y-8` outer rhythm as the rest of the page.

### Component file layout

```
web/src/components/Discovery/
├── DiscoverySection.tsx        # Container — handles enabled/errored/healthy states + REST seed + WS subscriptions
├── DiscoveryBanner.tsx          # Top banner (disabled / errored / healthy variants)

web/src/hooks/
```

### Visual hierarchy (top → bottom)

1. **Section header** — `<h3>` "Discovery Scanner" + status pill ("Scanner ON" / "Scanner OFF" / "Scanner errored") + last-run timestamp on the right
2. **State banner** (conditional) — only shown when disabled OR errored; hidden when healthy
3. **Cycle Stats card** (full-width, 7-tile horizontal grid; collapses to 2-col on tablet ≤768px)
4. **2-column row** on desktop ≥1024px:
   - **Left (60%):** Score History card with symbol-selector + Recharts line chart + threshold band overlay
   - **Right (40%):** Why Rejected breakdown table
   - **Tablet/mobile:** stack vertically
5. **Promote/Demote Timeline placeholder card** (full-width, low-emphasis, dotted border to signal "future content")

## Interaction & Data Contract

### REST seed (initial mount)

| Endpoint | Response shape | Used by |
|----------|---------------|---------|
| `GET /api/pg/discovery/state` | `{ ok: true, data: { enabled, last_run_at, next_run_in, candidates_seen, accepted, rejected, errors, duration_ms, why_rejected: { reason: count }, score_snapshot: [{symbol, score, sub_scores, gates_passed, gates_failed}] } }` | `usePgDiscovery` hook on mount |
| `GET /api/pg/discovery/scores/{symbol}` | `{ ok: true, data: { symbol, points: [{ts, score, sub_scores}], threshold_band: {auto_promote: number} } }` | `ScoreHistoryCard` when user selects a symbol |

Standard envelope `{ ok, data?, error? }` per `CLAUDE.local.md` — verified against existing `internal/api/handlers.go` `writeJSON` helper.

### WebSocket subscriptions (incremental updates)

| Channel | Trigger | UI update |
|---------|---------|-----------|
| `pg_scan_cycle` | Every scanner cycle completes | Refresh CycleStatsCard + WhyRejectedCard atomically |
| `pg_scan_metrics` | 5s throttle | Refresh CycleStatsCard "next run in" countdown only |
| `pg_scan_score` | Debounced (per symbol) | Append point to ScoreHistoryCard chart for currently-selected symbol; ignore if not selected |

Hook reuses the existing `usePriceGapWebSocket` pattern from `PriceGap.tsx:141–199` (parses newline-separated JSON, switches on `msg.type`). No new WS infrastructure.

### Loading states

- **Initial mount, REST in flight:** show skeleton (`bg-gray-800 animate-pulse rounded-lg` at fixed height — reuses existing skeleton pattern at `Analytics.tsx:82`).
- **Score history switching symbols:** show inline spinner (existing `text-xs text-gray-500` "Loading…" used elsewhere) inside the chart card; chart fades to 50% opacity until new data arrives.
- **WS reconnecting:** banner shows "Reconnecting…" using existing pattern from PriceGap.tsx (no new copy needed).

### Empty states (specific)

| Card | Empty trigger | Display |
|------|---------------|---------|
| Cycle Stats | `last_run_at == null` | Render the 7 tiles with "—" values + "Never" for last_run; do NOT render placeholder copy |
| Score History | `score_snapshot == []` | "No score history yet" copy + suggestion to set up universe (`pricegap.discovery.scoreHistory.noUniverse`) |
| Why Rejected | `Object.keys(why_rejected).length === 0` | `pricegap.discovery.whyRejected.empty` |
| Timeline Placeholder | always (in Phase 11) | `pricegap.discovery.timeline.placeholder` |

### Three scanner visual states (mandatory distinction)

| State | Trigger | Visual signal |
|-------|---------|---------------|
| Running (healthy) | `enabled=true`, no errors, `last_run_at` recent | Green "Scanner ON" pill in header; no banner; cycle stats render normally |
| Disabled | `enabled=false` | Gray "Scanner OFF" pill; gray disabled banner at top; cycle stats render with "—" / "Never" values (still visible — operator can read what would populate); banner copy explains config flag |
| Errored | `enabled=true` AND (`errors > 0` for last cycle OR `cycle_failed=true`) | Red "Scanner errored" pill; red error banner at top with error count + exchange list; cycle stats still rendered with whatever values came through |

**Verified against:** existing `colorMapNew` in `StatusBadge.tsx:9–19` (active=green, error=red, closed=gray).

## Accessibility Notes

- **Color contrast:** all text on dark canvas (`#0b0e11`) uses gray-100/300/400 which meet WCAG AA (≥4.5:1) on the existing palette; verified against existing dashboard which has been live since v0.20.x.
- **Focus indicators:** inherit existing `outline: 2px solid #f0b90b` from `index.css:230` — no override.
- **Keyboard navigation:** Score History symbol-selector is a native `<select>` (keyboard-accessible by default). No custom dropdown.
- **Screen reader:** banner uses `role="status"` (informational); errored banner uses `role="alert"` (live region) so screen readers announce scanner failures.
- **No motion:** no auto-playing animations beyond the existing pulse-loading skeleton; respects `prefers-reduced-motion` indirectly via Tailwind defaults.

## Copywriting tone & rules

- **Calm, not alarmist:** error banner says "Scanner reported errors in the last cycle" (descriptive), not "Scanner CRASHED" (alarming). Operators are running live trading — UI noise costs trust.
- **Operator language:** assume the reader is the bot operator (technical, English/Mandarin bilingual). Use config-key names verbatim (`PriceGapDiscoveryEnabled`, `PriceGapDiscoveryUniverse`) — operators search logs by key.
- **No marketing copy:** every label is a noun phrase or stat name. No "powerful", "smart", "intelligent" words.
- **Tokens use `{name}` not `{{name}}`:** matches existing `replaceTokens` helper in `PriceGap.tsx:131–137`.
- **zh-TW specifics:** use Traditional Chinese characters (繁體); avoid simplified mainland forms. Default reading direction is left-to-right (matches en).
