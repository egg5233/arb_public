# Phase 9 — UI Design Contract

> Visual and interaction contract for the Price-Gap dashboard page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> **Guiding principle:** This page is a lightweight extension of `web/src/pages/Positions.tsx`. It inherits every visual and interaction token already shipped in that file — there is no new design language. CONTEXT.md D-01..D-25 are locked and not re-opened.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn, no components.json) |
| Preset | not applicable |
| Component library | none — hand-rolled Tailwind primitives that already exist in `web/src/pages/*` |
| Icon library | none — Unicode glyphs (▶ ▼ 📝) and CSS-only indicators, matches existing codebase |
| Font | system default (`font-sans` default) + `font-mono` for numerics and symbols |
| CSS framework | Tailwind utility classes only, no `tailwind.config.js` — uses default Tailwind palette |
| Stack | React 18 + Vite + TypeScript, embedded into Go binary via `go:embed` |

**Inheritance rule:** New components **copy token choices from `Positions.tsx` verbatim**. If a token is not used there, this spec declares it explicitly below.

## Spacing Scale

Matches existing dashboard usage. Multiples of 4 only.

| Token | Tailwind | Value | Usage |
|-------|----------|-------|-------|
| xs | `gap-1` / `p-1` | 4px | Icon-to-text gaps, badge inner padding |
| sm | `gap-2` / `p-2` / `py-2` | 8px | Compact table cell padding, badge horizontal padding |
| md | `p-4` / `gap-4` | 16px | Card/table panel padding, section heading margins |
| lg | `space-y-6` / `p-6` | 24px | Stack gap between top-level sections on the page |
| xl | `space-y-8` | 32px | Page-level vertical rhythm between the 5 stacked sections |

**Row density:** Tables use `py-2` (8px vertical) per row with `text-sm` content — identical to `Positions.tsx` L132, L168. Dense by design per D-02.

**Inherited patterns (not re-declared here):** Badge pill classes are inherited verbatim from `web/src/pages/Positions.tsx` L374 (rotation badge). This spec does not re-declare those non-grid values; they are referenced as shipped patterns, not new declarations.

## Typography

Three sizes shipped in `Positions.tsx`. No new sizes introduced. **Exactly two font weights are used across the page.**

| Role | Tailwind | Size | Weight | Line Height |
|------|----------|------|--------|-------------|
| Page title | `text-xl font-bold` | 20px | 700 | 1.3 |
| Section heading | `text-sm font-bold` | 14px | 700 | 1.4 |

## Interaction Contracts

### Header toggles

- **Master enable switch** (`PriceGapEnabled`): matches existing config-tab toggle visual. Tooltip: `Master on/off for price-gap tracker. Persists to config.json.`
- **Paper toggle** (`PriceGapPaperMode`, D-07, D-12): **labeled `PAPER MODE`** in bold; when ON, shows violet pill and its background subtly glows `bg-violet-500/15`. When OFF, neutral gray. Tooltip on hover: `When ON, new entries are simulated. Existing open positions keep their original mode.`
- Both toggles POST to `/api/config`; optimistic update rolls back on `{ok:false}`.

### Manual disable (D-15)

Clicking `Disable` on an enabled candidate row opens an inline prompt (native `<dialog>` or the existing modal pattern at `Positions.tsx` L602):

```
Title: Disable BTCUSDT?
Body:  This candidate will be blocked from new entries.
       Existing open positions stay open.
Field: Reason (optional) — placeholder "manual"
Buttons: Disable (bg-red-600 text-white) | Keep active (bg-gray-700 text-gray-300)
```

On confirm, POST writes `pg:candidate:disabled:<symbol>` with `{reason: text || "manual", disabled_at: now}`.

### Re-enable confirmation (D-14)

Clicking `Re-enable` opens the same modal shell with:

```
Title: Re-enable {SYMBOL}?
Body:  Disabled {age} ago for: {reason}
       {supplemental detail if exec_quality: "realized X bps vs modeled Y bps"}
       Re-enabling clears the safety gate.
Buttons: Re-enable (bg-red-600 text-white) | Keep disabled (bg-gray-700)
```

Note: Re-enable uses the destructive-red button intentionally — it clears a safety gate (D-14).

### WebSocket update behavior

Per D-05/D-06:
- Mount fires REST seed → tables populated with skeleton (see Loading State).
- `pg_positions` topic replaces live-positions state snapshot.
- `pg_event` topic prepends to Closed Log (if exit event) or flashes an entry (optional subtle row highlight — 800ms `bg-green-500/10` fade for new entry rows).
- `pg_candidate_update` patches matching row by symbol.
- No toast system — log events surface in Closed Log and Telegram per D-18..D-22.

### Loading state (REST seed)

Per-section skeleton: one row of `<div className="h-4 bg-gray-800 animate-pulse rounded">` placeholders repeated 3 times inside each `<tbody>`. Matches the visual weight of real rows. **No full-page spinner**, no shimmer library. Skeleton replaced in-place when seed arrives.

### Error state

If REST seed fails: `<div className="text-red-400 text-xs p-4">Could not load {section}. Retry</div>` with a text-button `Retry`. If WS disconnects: subtle banner at top of the page `<div className="bg-yellow-500/10 text-yellow-400 text-xs px-4 py-2">Live updates paused — reconnecting…</div>` (auto-dismisses on reconnect).

## State Machine Contracts

### Paper/Live mode visual

| State | Header pill | Row tint (new entries) | Badge in Symbol col |
|-------|-------------|------------------------|---------------------|
| `PriceGapPaperMode=true` | `PAPER` (violet) | `bg-violet-500/5` | `PAPER` on every new row |
| `PriceGapPaperMode=false` | `LIVE` (green) | none | `LIVE` on every new row |

Rows keep the badge of the mode they were opened in (D-11). A row never changes its own badge mid-life.

### Candidate status state machine (visual)

```
Active (green) ──[Disable]──▶ Disabled (yellow, grayed row)
                                 │
                                 └─[Re-enable with confirmation]──▶ Active
```

Exec-quality auto-disable (D-19) transitions the row to Disabled server-side; the dashboard reflects it on next WS `pg_candidate_update`. The reason string surfaced in the row is the exact Redis value (per specifics.md "surface the exact disabled-reason string from Redis verbatim").

### Position row lifecycle (visual cue only)

- New entry: 800ms `bg-green-500/10` flash on the inserted row (non-blocking, cosmetic).
- Exit: row removed from Live Positions, row prepended to Closed Log with the same subtle flash.

## Data Persistence Decisions (locking Claude's Discretion)

From CONTEXT.md Claude's Discretion checklist — locked here for planner:

| Question | Locked answer |
|----------|--------------|
| Candidate `enabled` toggle — Redis-only or config.json? | **Redis-only** (`pg:candidate:disabled:<symbol>`). Matches Phase 8 D-19 / D-20 schema exactly. `pg-admin disable` and the dashboard must write the same key with the same JSON shape `{reason, disabled_at}`. |
| `PriceGapPaperMode` persistence | **config.json** via existing POST `/api/config` + `.bak` backup (D-12 already locks this). |
| REST seed endpoint shape | **Single aggregate** `GET /api/pricegap/state` returning `{enabled, paper_mode, budget, candidates[], active_positions[], recent_closed[], metrics[]}`. | Closed Log default page size | **100 rows**. `Load 100 more` action. Hard cap 500 to align with `pg:history` LIST cap from Phase 8 D-14. |
| Default sort — Live Positions | Open time desc (session-local, re-sortable). |
| Default sort — Candidates | Symbol alpha. |
