# Phase 9 ‚ÄĒ UI Design Contract

> Visual and interaction contract for the Price-Gap dashboard page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> **Guiding principle:** This page is a lightweight extension of `web/src/pages/Positions.tsx`. It inherits every visual and interaction token already shipped in that file ‚ÄĒ there is no new design language. CONTEXT.md D-01..D-25 are locked and not re-opened.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn, no components.json) |
| Preset | not applicable |
| Component library | none ‚ÄĒ hand-rolled Tailwind primitives that already exist in `web/src/pages/*` |
| Icon library | none ‚ÄĒ Unicode glyphs (‚Ė∂ ‚Ėľ ūüďĚ) and CSS-only indicators, matches existing codebase |
| Font | system default (`font-sans` default) + `font-mono` for numerics and symbols |
| CSS framework | Tailwind utility classes only, no `tailwind.config.js` ‚ÄĒ uses default Tailwind palette |
| Stack | React 18 + Vite + TypeScript, embedded into Go binary via `go:embed` |

**Inheritance rule:** New components **copy token choices from `Positions.tsx` verbatim**. If a token is not used there, this spec declares it explicitly below.

## Spacing Scale

Matches existing dashboard usage. Multiples of 4 only.

| Token | Tailwind | Value | Usage |
|-------|----------|-------|-------|
| xs | `gap-1` / `p-1` | 4px | Icon-to-text gaps, badge inner padding |
| sm | `gap-2` / `p-2` / `py-2` | 8px | Compact table cell padding, badge horizontal padding |
| md | `p-4` / `gap-4` | 16px | Card/table panel padding, section heading margins |
| lg | `space-y-6` / `p-6` | 24px | Stack gap between top-level sections on the page |
| xl | `space-y-8` | 32px | Page-level vertical rhythm between the 5 stacked sections |

**Row density:** Tables use `py-2` (8px vertical) per row with `text-sm` content ‚ÄĒ identical to `Positions.tsx` L132, L168. Dense by design per D-02.

**Exceptions:** Badge pill uses `px-1.5 py-0.5` (6px √ó 2px) ‚ÄĒ matches `Positions.tsx` L374 rotation badge; non-grid value is intentional for pill proportion.

## Layout Contract (Price-Gap page)

Per D-01 ‚ÄĒ single scroll with five stacked sections in fixed order. Wrapper: `<div className="space-y-8">`.

1. **Header / Status bar** ‚ÄĒ page title on the left, two toggles + status badges on the right (`flex items-center justify-between`).
   - Page title: `<h2 className="text-xl font-bold text-gray-100">Price-Gap</h2>`
   - Right cluster: master enable switch + Paper toggle (D-07), plus a compact stat strip (`Open: N ‚ÄĘ Budget: $X used / $Y`).
2. **Candidates section** ‚ÄĒ table, per D-13/D-15. Columns: Symbol, Long-exch, Short-exch, Threshold (bps), Max notional, Status, Disabled reason + time, Action (Disable / Re-enable).
3. **Live Positions section** ‚ÄĒ table, per D-03.
4. **Closed Log section** ‚ÄĒ table, per D-04. Default sort pinned to close-time desc.
5. **Rolling Metrics section** ‚ÄĒ table, per D-25.

Each section is wrapped in: `<div className="bg-gray-900 border border-gray-800 rounded-lg p-4 overflow-x-auto">` (matches `Positions.tsx` L127).

Section title above each panel: `<h3 className="text-sm font-semibold text-gray-300 uppercase tracking-wide mb-3">{t(...)}</h3>`.

## Interaction Contracts

### Header toggles

- **Master enable switch** (`PriceGapEnabled`): matches existing config-tab toggle visual. Tooltip: `Master on/off for price-gap tracker. Persists to config.json.`
- **Paper toggle** (`PriceGapPaperMode`, D-07, D-12): **labeled `PAPER MODE`** in bold; when ON, shows violet pill and its background subtly glows `bg-violet-500/15`. When OFF, neutral gray. Tooltip on hover: `When ON, new entries are simulated. Existing open positions keep their original mode.`
- Both toggles POST to `/api/config`; optimistic update rolls back on `{ok:false}`.

### Manual disable (D-15)

Clicking `Disable` on an enabled candidate row opens an inline prompt (native `<dialog>` or the existing modal pattern at `Positions.tsx` L602):

```
Title: Disable BTCUSDT?
Body:  This candidate will be blocked from new entries.
       Existing open positions stay open.
Field: Reason (optional) ‚ÄĒ placeholder "manual"
Buttons: Disable (bg-red-600 text-white) | Cancel (bg-gray-700 text-gray-300)
```

On confirm, POST writes `pg:candidate:disabled:<symbol>` with `{reason: text || "manual", disabled_at: now}`.

### Re-enable confirmation (D-14)

Clicking `Re-enable` opens the same modal shell with:

```
Title: Re-enable {SYMBOL}?
Body:  Disabled {age} ago for: {reason}
       {supplemental detail if exec_quality: "realized X bps vs modeled Y bps"}
       Re-enabling clears the safety gate.
Buttons: Re-enable (bg-red-600 text-white) | Cancel (bg-gray-700)
```

Note: Re-enable uses the destructive-red button intentionally ‚ÄĒ it clears a safety gate (D-14).

### WebSocket update behavior

Per D-05/D-06:
- Mount fires REST seed ‚Üí tables populated with skeleton (see Loading State).
- `pg_positions` topic replaces live-positions state snapshot.
- `pg_event` topic prepends to Closed Log (if exit event) or flashes an entry (optional subtle row highlight ‚ÄĒ 800ms `bg-green-500/10` fade for new entry rows).
- `pg_candidate_update` patches matching row by symbol.
- No toast system ‚ÄĒ log events surface in Closed Log and Telegram per D-18..D-22.

### Loading state (REST seed)

Per-section skeleton: one row of `<div className="h-4 bg-gray-800 animate-pulse rounded">` placeholders repeated 3 times inside each `<tbody>`. Matches the visual weight of real rows. **No full-page spinner**, no shimmer library. Skeleton replaced in-place when seed arrives.

### Error state

If REST seed fails: `<div className="text-red-400 text-xs p-4">Could not load {section}. Retry</div>` with a text-button `Retry`. If WS disconnects: subtle banner at top of the page `<div className="bg-yellow-500/10 text-yellow-400 text-xs px-4 py-2">Live updates paused ‚ÄĒ reconnecting‚Ä¶</div>` (auto-dismisses on reconnect).
