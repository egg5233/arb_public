<objective>
Implement PG-OPS-04: paper mode runs full event detection + risk gate + entry/exit logic, but synthesizes fills instead of calling `ex.PlaceOrder`. Single chokepoint pattern per Research Pattern 2; no duplicated monitor/exit paths.

Purpose: Operators run a 3-day paper validation window before deploying capital. Every codepath must be exercised end-to-end (including slippage logging) so the Closed Log rows carry realistic Modeled/Realized/Delta values (Pitfall 7 — synthesize at mid±modeled/2, not at mid).

Paper-fill synthesis strategy (LOCKED — do not defer):
- Buy leg synth price = `fillPrice + fillPrice * (modeledEdgeBps / 2.0) / 10_000.0` (adverse above mid)
- Sell leg synth price = `fillPrice - fillPrice * (modeledEdgeBps / 2.0) / 10_000.0` (adverse below mid)
- `modeledEdgeBps` derived from the detection snapshot's `ModeledSlipBps` value already computed by Phase 8 detector (field populated on PriceGapPosition at L142 of execution.go).
- Rationale: produces non-zero realized slippage that exercises the full pipeline; Closed Log shows meaningful Modeled/Realized/Delta numbers; operator gains trust before live capital.

Output: Paper branch + Mode stamping + tests. Live path unchanged.
</objective>
