<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Unauthenticated → /api/pricegap/* | Must go through s.cors(s.authMiddleware(...)) — any bypass is a high-severity leak |
| Client body → Redis | Reason string crosses into Redis; must be length-capped and control-char-stripped |
| /api/config writer → config.json | Paper-mode flip routes through the single authorized writer with .bak backup |
| Symbol param → Redis key | Must be allowlisted against cfg.PriceGapCandidates to prevent arbitrary key writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-09-06 | Spoofing | Unauthenticated /api/pricegap/state | high | mitigate | All 7 routes registered through s.cors(s.authMiddleware(...)); acceptance test verifies 401 without bearer |
| T-09-07 | Tampering | Arbitrary Redis key write via {symbol} path param | high | mitigate | Symbol allowlisted against cfg.PriceGapCandidates before SetCandidateDisabled; unknown → 400 |
| T-09-08 | Tampering | Reason-string injection into Telegram body (downstream) | medium | mitigate | Length cap 256 chars + control-char strip at API entry; Telegram formatter in Plan 04 also escapes |
| T-09-09 | Elevation | Unauthorized paper→live flip | high | mitigate | paper_mode only writable via /api/config (bearer-gated); no separate unauthed endpoint |
| T-09-10 | Information Disclosure | Position payload contains API secrets | low | mitigate | PriceGapPosition schema has no *Key/*Secret/*Token fields (verified Plan 01); WS broadcast uses same struct |
| T-09-11 | Repudiation | CLI disable vs dashboard disable value drift (Pitfall 6) | medium | mitigate | Single writer SetCandidateDisabled; acceptance test verifies pg-admin calls the same store method |
| T-09-12 | Denial of Service | /api/pricegap/closed with limit=1_000_000 | medium | mitigate | Handler caps limit <= 500 (matches Phase 8 pg:history LIST cap) |
</threat_model>
