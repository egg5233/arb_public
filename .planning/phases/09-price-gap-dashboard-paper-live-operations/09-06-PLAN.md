Wire tracker events to the dashboard WebSocket hub and the Telegram notifier. This is where Plans 01–05 converge: Plan 01 gave us the Broadcaster seam, Plan 02 implemented it on *Server, Plan 03 added Mode stamping, Plan 04 added Notify methods, Plan 05 added metrics. Output: Hook invocations at 5 well-defined call sites + cmd/main.go DI + tests.
</objective>

2. **closePair success** (monitor.go): `t.broadcaster.BroadcastPriceGapEvent(PriceGapEvent{Type: "exit", Position: pos, Reason: reason}); t.notifier.NotifyPriceGapExit(pos, reason, pnl, duration)`
3. **risk gate denial** (risk_gate.go per-gate return path): `t.notifier.NotifyPriceGapRiskBlock(symbol, gate, detail)` (no broadcast — UI surface via REST + candidate_update only)
4. **exec-quality auto-disable** (slippage.go recordSlippageAndMaybeDisable): `t.broadcaster.BroadcastPriceGapEvent(PriceGapEvent{Type: "auto_disable", Symbol: symbol, Reason: "exec_quality"}); t.broadcaster.BroadcastPriceGapCandidateUpdate(PriceGapCandidateUpdate{Symbol: symbol, Disabled: true, Reason: "exec_quality ...", DisabledAt: time.Now().Unix()}); t.notifier.NotifyPriceGapRiskBlock(symbol, "exec_quality", detail)`
