<objective>
PG-FIX-02: Audit-first diagnose the v2.0 UAT auto-POST that flipped `paper_mode=false` on dashboard page load, then install a server-side chokepoint guard at `/api/config` that rejects any write to `price_gap_paper_mode` (flat or nested `price_gap.paper_mode`) unless the request body carries `operator_action: true`. Page-load hydration and generic config persists get HTTP 409.

Purpose: Belt-and-suspenders defense. Audit (D-05) establishes the negative even if the bug doesn't reproduce; server guard (D-06) prevents the entire failure mode going forward. Phase 9 `pos.Mode` per-position immutability is untouched — this guard sits at the engine-wide config-write layer.

Output: Audit doc + HAR capture, handlers.go guard + JSON field, test coverage (reject without marker for flat + nested write paths, accept with marker, GET does not mutate, existing tests updated to include marker), VERSION + CHANGELOG bump.

NOTE: This plan is server-side ONLY. The `web/src/pages/PriceGap.tsx` `togglePaper` wire-up (sending `operator_action: true`) is delivered by Plan 04 (Wave 2) along with the Configuration card. Plan 04 `depends_on: [16-02]` so the server guard is in place when the client edit ships. Until Plan 04 lands, the existing dashboard `togglePaper` POST will be 409-rejected — operators are notified via CHANGELOG that paper-mode toggling temporarily requires pg-admin until Plan 04 ships in the same release window.
</objective>
