## Interaction Contracts

### Configuration card collapse / expand

- Default state: collapsed (operators visit page primarily for status, not config).
- Click anywhere on the card heading row toggles expand/collapse — full row is the click target (44px min height per accessibility touch-target convention; matches existing card heading pattern in `RampReconcileSection`).
- Expanded state persists in `localStorage` under key `arb_pg_config_expanded` (operator preference, not security-relevant).
- Aria: heading has `aria-expanded` + `aria-controls` pointing to the content panel.

### Per-subsection save flow

- Each subsection has its own "Save {section}" button (per-subsection save reduces blast radius of mistakes; matches existing `Config.tsx` per-section save pattern).
- Save button enabled only when at least one field is dirty (`isDirty` state per section).
- Save button shows spinner + "Saving…" label during in-flight POST.
- On 200: success toast, fields reset to clean (server values).
- On 409: error toast referring to operator-action requirement (paper-mode toggle path only).
- On 422 / validation error: inline error under offending field, save button re-enabled.
- On 500 / network: retry-friendly error toast.

### Typed-phrase modal flow (ENABLE-LIVE-CAPITAL / ENABLE-BREAKER)

Reuse `BreakerConfirmModal` interaction verbatim (Phase 15 contract):

1. Operator clicks toggle in OFF state.
2. Modal opens; toggle visual stays OFF until confirmation succeeds.
3. Real-trip warning banner shown when toggling to less-safe direction (live ON / breaker disable).
4. Operator types magic phrase exactly (case-sensitive, no trim).
5. Confirm button enables.
6. Click confirm → POST through `postConfig` with field + `operator_action: true` for paper-mode-related guard.
7. Server validates phrase server-side per CONTEXT D-21 (defense in depth).
8. On success: modal closes, toggle visual updates, success toast.
9. On reject: error renders inline in modal; modal stays open; phrase input stays filled.

### Inline validation (number inputs)

- HTML5 `type="number"` with `min`/`max`/`step` attributes set per D-17 ranges.
- On blur: clamp display value to range; flag with red border + `text-xs` error if out of range (matches `BreakerConfirmModal:140-144` error pattern).
- Server-side validation is authoritative (existing `validatePriceGapLive` covers all fields per RESEARCH.md).

### Paper-mode toggle (PG-FIX-02 wire-up)

- Existing button visual stays unchanged.
- POST body: `{ price_gap_paper_mode: <next>, operator_action: true }`.
- On 409 reject: revert local state + show toast `pricegap.config.paperMode.serverReject`. Toast message tells operator the toggle button MUST be the click source (helps the rare case where some background hydration tries to flip).

### Keyboard navigation

- Tab order: card-heading → subsection-heading → first-field-in-section → next-field → save-button → next-section.
- Esc dismisses any open modal (when not busy).
- Enter inside a typed-phrase input does NOT submit — operator must explicitly click confirm (matches Phase 15 — defense against pasted-phrase auto-submit). Phase 16 must follow this.
