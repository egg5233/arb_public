## Design System

| Property | Value |

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (normal) | 1.5 | `text-sm` |
| Label | 12px | 400 (normal) | 1.5 | `text-xs` |
| Heading | 20px | 700 (bold) | 1.2 | `text-xl font-bold` |

**Note:** The project uses exactly 3 type sizes across the dashboard. Config field labels use `text-xs`, field values and table content use `text-sm`, and page/section headings use `text-xl font-bold`. Visual distinction between body and label is achieved by the size difference (12px vs 14px) and muted color (`text-gray-400`), not by font weight. Monospace (`font-mono`) is applied to all numeric values, rates, and financial data.

**Source:** Codebase inspection -- consistent across all page components.

### Config Field Rendering Pattern

Each new toggle follows the established pattern in `renderSfExitTab()`:

```
<div className="flex items-center justify-between">
  <div>
    <span className="text-sm text-gray-100">{t('cfg.sf.fieldLabel')}</span>
    <Tooltip text={t('cfg.sf.fieldLabelDesc')} />
  </div>
  <ToggleSwitch on={value} onChange={handler} />
</div>
```

Each new number field follows the established `NumberField` pattern:

```
<NumberField
  label={t('cfg.sf.fieldLabel')}
  tooltip={t('cfg.sf.fieldLabelDesc')}
  path={['spot_futures', 'json_field_name']}
  config={config}
  onChange={handleChange}
/>
```

### Conditional Visibility

When a feature toggle is OFF, its associated threshold field should be **visually muted** (`opacity-50`) but remain visible. This matches the existing pattern for `borrow_spike_enabled` in `renderSfExitTab()` where related fields are shown but dimmed when the parent toggle is off.

## Interaction Contracts

### Config Toggle Behavior

1. All new toggles default to OFF (project convention: `feedback_risk_configurable_switch.md`).
2. Toggling ON immediately enables the feature for the next scan/monitor cycle.
3. Changes persist via POST `/api/spot/config` and survive restart via Redis.
4. No confirmation dialog for toggle changes (matches existing toggle behavior).
5. Number field changes apply on blur or Enter key (existing `NumberField` behavior).

### Opportunities Page Data Source Indicator

When the native scanner is active, display a small `text-xs text-gray-500` data source label in the spot-futures summary bar:

```
[green dot] 3 Actionable  [gray dot] 12 Filtered  [label] Source: Native
```

This reuses the existing summary bar pattern at `Opportunities.tsx:193`. The source label uses `opp.source` field from `SpotOpportunity`.

### No New Pages, Modals, or Navigation Items

Phase 2 does not add any new pages or navigation items. All changes are within existing Config.tsx and Opportunities.tsx pages.
